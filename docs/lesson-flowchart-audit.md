# Lesson Flowchart Audit — 2026-06-02 (placeholder)

> Static triage of every Mermaid source attached to a CurriculumLesson.
> Generated by `scripts/audit-flowcharts.mjs`. Re-runnable. No content is mutated.

## How to populate this report

The audit script needs DB access. Run it from the `packages/db` directory so
`@prisma/client` resolves correctly:

```bash
cd packages/db && node ../../scripts/audit-flowcharts.mjs
```

It writes (and overwrites) two files:

- `docs/lesson-flowchart-audit.md` — this file, with live scoring tables and
  the top-10 detail section.
- `docs/lesson-flowchart-audit.json` — raw scoring data for follow-up
  automation (e.g. feeding the worst offenders to a fix-agent).

The script reads `.env` from the repo root and `packages/db/.env`, in that
order, with the first-set-wins precedence. No CLI flags are required.

## What the script does

For every `CurriculumLesson` it pulls mermaid sources from **four** places:

1. `flowchartMermaid` column (dedicated)
2. `mindmapMermaid` column (dedicated)
3. ` ```mermaid ` fenced blocks inside `contentEnMdx`
4. ` ```mermaid ` fenced blocks inside `contentArMdx`

Each block is scored independently. A block is flagged when `total ≥ 15`.

### Scoring

| heuristic | formula | rationale |
|---|---|---|
| nodeCount | `max(0, (n − 12) × 2)` | clutter starts at ~12 nodes |
| lineCount | `max(0, (n − 20) × 1.5)` | long sources are usually checklists, not diagrams |
| unquotedParensInSquares | `n × 5` | `[Label (qualifier)]` parses as a different shape — guaranteed render breakage |
| unquotedSpecialChars | `n × 2` | `&`, `%`, `/`, `:`, em-dashes inside labels |
| disconnectedSubgraphs | `(n − 3) × 4` if n > 3 | nested rectangle soup, curve='basis' routes it poorly |
| noOrientation | `+3` | default top-down rarely matches intent; produces tall columns |
| hasUnusualType | `+5` | sequenceDiagram / gantt / classDiagram don't fit the curriculum aesthetic and bypass the recent font-size lift |

### Output structure

The populated report has:

1. **Systemic issues** banner — patterns that hit a large fraction of the
   corpus, with a recommended fleet-wide fix (e.g. "prepend `flowchart LR`
   to all undeclared blocks").
2. **Score distribution** histogram (0–5, 5–10, 10–20, 20+).
3. **Chart type tally** — how many flowcharts vs sequenceDiagrams etc.
4. **Heuristic hit counts** — which heuristic flagged how many blocks.
5. **Top 50 worst offenders** table — ranked by total score, with
   `slug`, `titleEn`, `origin`, and `flags`.
6. **Top 10 detailed breakdown** — per offender:
   - score breakdown
   - the mermaid source verbatim
   - why it's flagged (prose)
   - top 5 get a concrete suggested rewrite (single chart cleanup OR
     a split into multiple smaller flowcharts), rest get a brief fix tip
7. **Method** — the formula above, restated.

## Action plan once populated

1. Read the **Systemic issues** section first — if 60% of blocks are missing
   `flowchart LR`, that's a one-liner cleanup across the corpus and is worth
   doing before touching individual charts.
2. Pull the **Top 10 detailed breakdown** into a fix-agent ticket. The
   suggested rewrites for the top 5 are structural; review them against the
   surrounding lesson prose before committing.
3. Re-run the audit after a fix batch. Score totals should drop and the
   flagged count should shrink.
4. Once flagged count is < 5% of blocks, declare the audit complete and
   archive this report.
